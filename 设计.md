# ibi.ren (Idol Bank IP) 一期产品需求文档 (PRD) 暨资产交付规范

**文档版本:** V 1.0 (抢跑公测版)

**目标愿景:** 构建中国首个标准化的 AI 虚拟人资产与版权确权、交易、分发中心。一期核心目标是跑通首批 100 个自营/内测形象的展示、确权存证、防盗图与阶梯式意向授权交易。

## 🟢 第一部分：三层架构详细 PRD

### 1. 表现层 (Presentation Layer) —— 前台门户与工作台

**产品目标：** 打造类似“高端模特经纪公司”的视觉体验，同时通过严格的预览机制防止 B 端客户恶意白嫖（盗图、垫图）。

#### 1.1 前台选角门户 (Portal)

- **形象瀑布流货架:**
  - **筛选与标签:** 支持按属性筛选，如【性别】(男/女/无性别)、【视觉年龄】(童/青/中/老)、【风格】(现代/古风/赛博/二次元)、【应用场景】(短剧群演/品牌代言/平面模特)。
  - **列表页防盗:** 列表页缩略图强制压缩分辨率（如 600x600），并叠加平台显性半透明水印。
- **AI IP 详情页 (Detail Page):**
  - **盲盒式展示:** 采用视频或动图优先的展示策略（视频极难被AI工具提取高清正脸）。核心高清三视图只展示局部或打满高强度水印。
  - **资产包清单透出:** 清晰展示该 IP 包含哪些交付件（如：带 LoRA、带三视图、带设定），但不可下载。
  - **交易入口 (阶梯定价):** * 按钮 A：支付意向金（如 199元），获取带水印的非商用测试模型包（有效期7天）。
    - 按钮 B：发起正式版权授权采购（对接商务或线上直接电子签）。

#### 1.2 用户工作台 (Workspace)

- **A端 (创作者) 中心:** * 一期主要为后台导入。预留功能：实名认证 (KYC，对接阿里云/腾讯云活体检测)、资产打包上传向导（限制文件格式与大小）。
- **B端 (采购方) 中心:** * **我的资产库:** 查看已购买/预定的 IP 列表。
  - **合规材料下载:** 下载已签名的电子版《授权证书》及上链存证报告。
  - **安全交付机制:** 点击下载资产时，系统通过对象存储 (OSS) 实时生成带有时间限制的临时下载防盗链（Token 过期失效）。

### 2. 业务逻辑层 (Business Logic Layer) —— 交易与授权中枢

**产品目标：** 串联人、货、钱、合同，实现业务闭环。

#### 2.1 阶梯式交易引擎

针对抢跑期官方版权尚未下发的痛点，设计动态订单状态：

- **预售/意向金模式:** 买家支付小额押金，锁定某形象在某部短剧中的独占排他期。
- **附条件生效机制:** 系统生成订单时，标记“版权办理中”。平台提供担保承诺，若因版权问题导致 B 端损失，触发自动全额退款及理赔流程。

#### 2.2 自动化合同与签章工厂

- 根据 B 端选择的授权类型（单次广告 / 单部短剧 / 全网3年），系统调取对应的 Word/PDF 模板。
- 自动将 A 端创作者信息、B 端企业信息、IP 缩略图、授权期限合成到合同中。
- 调用第三方电子签平台 API（如法大大），强制要求 B 端在线完成企业签章后，方可解锁模型下载链接。

### 3. 资产与版权数据层 (Data & Backend Layer) —— 中后台与合规基建

**产品目标：** 沉淀高价值数据，建立坚不可摧的法律与技术护城河。

#### 3.1 两阶段确权状态机

每个 IP 资产在数据库中拥有严格的状态流转：

1. `[待审核]`：已上传，等待机器鉴黄鉴暴及人脸查重。
2. `[审核通过-存证中]`：调用区块链接口，将原始图片 Hash 上链。
3. `[公示与意向期]`：获得区块链存证证书，前台可展示、可售卖预售权，等待软著下发。
4. `[正式版权生效]`：运营人员后台填入国家版权局下发的作品登记号，转为可全量商用状态。

#### 3.2 隐形数字水印引擎 (防溯源利器)

- 创作者上传高清原图后，后端触发异步任务。
- 采用 **DWT-SVD（离散小波变换）** 或类似算法，将创作者 ID 和平台时间戳作为不可见暗水印写入图片像素深处。
- **业务价值：** 即使 B 端用户拿截图去“垫图重绘”，生成的衍生图极大概率依然保留该暗水印。一旦发生纠纷，平台可提取水印作为呈堂证供。

## 🔵 第二部分：Idol Bank 资产包交付规范 (Asset Pack Spec)

为了让短剧制作公司、广告公司“拿来即用”，ibi.ren 平台拒绝只卖单张好看的 AI 生成图。每个上架的 IP 必须是一个 **“标准生产力工具包”**。

**资产包总大小限制：** 不超过 1 GB

**压缩格式：** 统一使用 `.zip` 打包

### 模块 1：Visual Matrix (核心视觉图谱) - 必选

这是 B 端客户做角色参考、海报物料的基础。

- **三视图 (Character Sheet):** 必须包含同一角色穿着基础服装的【正面】、【侧面 45 度】、【背面或正侧面】。分辨率不低于 `2048 x 2048`，格式为 JPG 或 PNG。
- **表情矩阵 (Expression Grid):** 至少包含 5 种标准表情图：喜、怒、哀、惊、自然平静。
- **立绘图 (Transparent Render):** 至少 1 张已去除背景的该角色全身或半身高清 PNG 格式图（带 Alpha 通道），方便 B 端直接拖入排版软件使用。

### 模块 2：AI Core (AI 生产力文件) - 必选

这是核心价值所在，决定了 B 端能否在自己的工作流中一致性地重现该角色。

- **模型文件 (LoRA / LyCORIS):** * **格式:** 必须为 `.safetensors` 格式（拒绝 `.ckpt`，防范恶意代码注入）。
  - **兼容性声明:** 必须明确声明训练底模（如：基于 SD 1.5 还是 SDXL 训练），文件大小建议控制在 300MB 以内。
- **生产说明书 (Recipe / Prompt Guide):**
  - **触发词 (Trigger Words):** 激活该角色的一组或多组专属 Tag。
  - **推荐参数:** 明确给出推荐的 LoRA 权重范围（如 `0.6 - 0.8`）、推荐的采样器 (Sampler) 及步数 (Steps)。
  - **测试样板:** 提供至少 3 张测试出图的完整 Prompt（正向与反向提示词）。

### 模块 3：Lore & Identity (人物设定与灵魂) - 必选

赋予模型商业化的人设价值，方便剧组选角。

- **基础档案 (`bio.txt` 或 `bio.md`):** * 【姓名】（平台代号或暂定名）
  - 【视觉年龄】
  - 【性格标签】（如：腹黑、阳光、清冷、病娇，不少于3个）
  - 【人物小传】（200-500字的背景故事，如：归国财阀千金、古代流浪剑客）。
- **声音资产 (Voice Reference) - [一期可选，后续必选]:**
  - 推荐的音色描述（如：成熟御姐音，带微哑）。
  - 可选附带一段 10-15 秒的 `.wav` 格式纯净干音文件，授权买家可将其投入 TTS（文本转语音）或声音克隆大模型中训练专属配音。

### 模块 4：Legal & Compliance (合规文件) - 仅平台后端可见

- **原始工程截图:** 创作者需提供训练该 LoRA 时，部分数据集的缩略图或 ComfyUI/WebUI 的工作流截图，证明其拥有底层生产过程（非盗取他人模型）。
- **免责电子签:** 创作者在工作台点击提交前，系统强制弹出的《原创声明及侵权责任自担协议》（留存 IP 与实名信息）。
