## 1. 4 层 AI 自动评分架构(用户原话)

### Layer 1 — 技术质量
- 执行者:ffprobe / opencv / ffmpeg(确定性工具)
- 覆盖度:100% 自动化
- 检查项:
  - 视频:分辨率 / 码率 / 编码 / 色彩空间 / 时长 / 帧率
  - 音频:电平 / 黑帧静帧 / 字幕同步
  - 3D:面数 / 法线 / UV / 骨骼完整性

### Layer 2 — 美学质量
- 执行者:VLM(Claude 3.5 Sonnet / GPT-4V / Qwen2-VL / GLM-4V)+ 视觉质量模型
- 覆盖度:60-95% 自动化
- 检查项:
  - 构图 / 色彩和谐 / 光照一致性 / 运镜平滑
  - 数字人口型同步(Wav2Lip/SyncNet)/ 表情自然度
  - 整体美感评分:DOVER / FAST-VQA / BVQA

### Layer 3 — 合规质量
- 执行者:NLP + 词表 + 内容安全 API(阿里云/腾讯云)
- 覆盖度:80-100% 自动化
- 检查项:
  - 广告法 / 平台规则 / IP 形象相似度 / 品牌元素
  - 形象授权 / 未成年人 / 政治敏感
- ⚠ **闸门规则**:任一不合规 → 总分 = 0(一票否决)

### Layer 4 — 商业价值
- 执行者:LLM 比对 brief 描述 + VLM 看内容
- 覆盖度:50-75% 自动化
- 检查项:
  - Hook 强度 / 信息传递完整性 / 目标人群匹配
  - CTA 清晰度 / 情感共鸣度 / 品牌调性

### 最终评分公式
```
总分 = L1 × 0.15 + L2 × 0.30 + L3 × 0.25 + L4 × 0.30

闸门:
- L3 任一不合规 → 总分 = 0
- L1 < 0.60 → 总分自动 < 0.50(直接打回)
- L4 < 0.40 → 触发"商业价值不达标"标记

分级:≥ 0.85 = S / 0.70-0.85 = A / 0.60-0.70 = B / < 0.60 = C
```

**关键设计**:每条扣分必须带 evidence(frame:timecode / bbox / OCR 文本 / 截图),不能黑盒给分。

## 2. 调研任务清单(2 周,A 选项)

### 2.1 VLM 美学评分 benchmark(W2.5 第 1-5 天)

| 待测 | 维度 | 备注 |
|---|---|---|
| Claude 3.5 Sonnet | 美学 / 构图 / 商业价值 | 需校准集 |
| GPT-4V | 同上 | 价格高 |
| Qwen2-VL | 同上 | 国产 + 中文短视频更优? |
| GLM-4V | 同上 | 同上 |

**校准集**:50-100 条真实 AIGC 短视频(自家生成 / 公开数据集 AVA / KoNViD-1k / LIVE-VQC)

**评估指标**:PLCC / SRCC / Kendall(与人类评分相关性)

### 2.2 视频质量学术模型(W2.5 第 3-7 天)

| 模型 | 来源 | 是否维护 | 备注 |
|---|---|---|---|
| DOVER | ICCV 2023 | 活跃 | 分离 aesthetic + technical |
| FAST-VQA | ECCV 2022 | 活跃 | 速度快 |
| BVQA | 2024 | 活跃 | 通用 |
| NIMA / MUSIQ / MANIQA / AesFormer | 图像美学 | 图像域 | 视频需扩展 |

**输出**:实际跑通 benchmark,出准确率对比表

### 2.3 内容安全 API 对比(W2.5 第 5-8 天)

| API | 提供方 | 覆盖 | 费用 | 误杀率 |
|---|---|---|---|---|
| 阿里云内容安全 Green | 阿里云 | 广告法 / 涉政 / 暴恐 / 色情 | 按量,需询价 | 待实测 |
| 腾讯云天御 | 腾讯云 | 同上 | 同上 | 同上 |
| 自建 NLP + 词表 | 自建 | 广告法 / 敏感词 | 低 | 误杀可控,但漏判多 |

**决策点**:直接用云 API 还是自建兜底层

### 2.4 数字人口型同步 SOTA(W2.5 第 7-9 天)

- Wav2Lip / SyncNet / VideoReTalking / DINet / MuseTalk
- 延迟 / 准确率 / 中文支持
- 是否能跑在 ECS(算力 / GPU?)

### 2.5 ECS 部署 + 成本测算(W2.5 第 9-12 天)

- ffprobe + ffmpeg:本地并发能力
- VLM 调用:Claude 约 ¥0.5/视频,跑 100 条/天成本
- 是否需要 GPU(3D 拓扑 / 视频美学模型)
- 阿里云 GPU 实例价格 vs 腾讯云 vs 自建

### 2.6 评分校准集获取(W2.5 第 10-14 天)

- 自家 seed 数据?
- 公开数据集(KoNViD-1k / LIVE-VQC / AVA 等)
- 邀请 5-10 位运营人工打分,做"人评 vs AI 评"对比

## 3. 业界对标(简述)

| 平台 | 评审机制 | 我们的差异化 |
|---|---|---|
| Synthesia / HeyGen | 内部 CV + 人工,无客观标准 | 我们做"标准" |
| TopYa / Pika / Runway | 无评审 | 我们有 marketplace 闭环 |
| Midjourney / Sora | 只做生成侧 | 我们做平台审核 |
| Fiverr / Upwork | 纯人工 | 我们 AI 评分 + 人工申诉 |
| 抖音 / 小红书 | 机器 + 人工抽审 | 我们给 AIGC 创作者做 |

**蓝海**:没有公开成熟的 AI 作品质量评分体系 — **蓝海也是难处**

## 4. 风险与缓解(已识别)

| 风险 | 缓解 |
|---|---|
| VLM 美学偏见(西方 vs 中国短视频审美) | 校准集 + 国产 VLM 对比 |
| 创作者博弈(刷指标) | evidence 必现 + 人工抽审 + 信用分联动 |
| 评分成本(每条 ¥0.5+) | 分层调用:L1 全跑,L2-L4 仅 dispute 时跑 |
| "AI 评 AI"合法性 | evidence 必现 + 人工申诉 + 评分公开可解释 |
| 评分漂移(VLM 升级后) | model_version 写入 audit |

## 5. 三个选项对比

### A — 深度调研 2 周(本文档目标)
- 跑 VLM benchmark,验证 DOVER/FAST-VQA/BVQA
- 调研阿里云/腾讯云内容安全
- 出 `docs/research/quality-eval-benchmark-2026.md`
- **结论导向**:有数据后再决定 W3 怎么落地

### B — 最小可用版 W3 上线
- Layer 1(ffprobe+ffmpeg,2天)+ Layer 3(阿里云内容安全+广告法词表,3天)
- **不做 Layer 2/4**
- 风险:美学 + 商业价值维度缺位,买家"AI 评分没用"感强

### C — 外部合作
- 清华 / 中科院 / 商汤联合实验室
- 投入 ¥300k+/半年
- 抢"行业标准制定者"位置

---

## 6. 我的建议

**先 A 后 B**:2 周深度调研出真实数据,再决定 B 的技术选型。直接做 B 会埋雷(用错 VLM / 选错 benchmark,半年后才发现准确率不够)。
